# OctoAcme — Project Management Processes (Overview)

OctoAcme runs projects with a customer-first, iterative approach that starts with a lightweight Project One-pager during initiation and moves into planning and execution only when success metrics, stakeholders, and team availability are clear. Planning breaks validated initiatives into shippable increments with clear acceptance criteria, estimates, and a Definition of Done. Core artifacts—project one-pager, backlog, release plan, and risk register—are kept as the single source of truth in the project repository.

Day-to-day workflows use a simple project board lifecycle (Backlog → Ready → In Progress → In Review → QA → Done) and a disciplined pull request process: small PRs where practical, acceptance criteria and issue links in PR descriptions, and automated CI (tests, linting, security scans) before requesting review. Sprint planning is timeboxed, prioritizes items that meet the Definition of Done, and documents cross-team dependencies so they can be tracked and escalated.

Roles and responsibilities are explicitly defined to ensure clear ownership. Product Managers own outcomes and success metrics and prioritize the backlog. Project Managers coordinate delivery, schedule, and risks. Developers implement features and tests, QA validates acceptance criteria (automated and manual where needed), and stakeholders provide alignment and approvals. These personas are documented to support consistent decision-making and to reduce single-person dependencies.

Quality assurance is multi-layered: unit tests for new logic, integration tests where applicable, end-to-end smoke tests for critical flows, security scanning in CI, and manual QA for feature acceptance when necessary. Communication cadence includes daily standups for blockers, weekly delivery syncs and PM/PdM alignments, regular demos at sprint or milestone end, and monthly stakeholder updates. Risks and incidents are tracked in a risk register and communicated via templates and a clear escalation path (team → PM → Product Lead → Sponsor), with blameless retrospectives and tracked action items to drive continuous improvement.
